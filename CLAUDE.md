# Novels

多本小說的工作區。一本書一個資料夾，**各書的文風規範寫在該書自己的 CLAUDE.md，不要寫進這一份**——混在一起會讓不同書的聲音互相污染。

## 目錄

- `_shared/style/` — 跨書共用的範文（文風正例）
- `_shared/templates/book/` — 新書骨架，開新書時整包複製出去
- `<代號>/` — 每本書

新書用**短代號**當目錄名，不要用書名。書名幾乎一定會改，目錄跟著改名會讓 git 的檔案歷史斷掉，之後追溯改稿過程會很痛。書名寫在該書 CLAUDE.md 開頭。

## 工作流程

- **一次只寫一章或一場戲**，寫完停下來等我審過再繼續。連寫多章會開始自我複製。
- 動筆前先讀該書的 `bible/` 和 `outline/`。
- 寫完把新增的設定寫回 `bible/`，新埋的伏筆記到 `notes/`。這兩個目錄是跨工作階段唯一的記憶，不寫回去就等於沒發生。
- 每章定稿後 commit，訊息格式：`<代號>: 第 N 章 <一句話重點>`。

## 文風

以各書 CLAUDE.md 裡的正例段落為準。**不要靠禁令式指示**（「不要寫得像 AI」「少用排比」）——那通常只是換一套同樣呆板的寫法。要調整文風就去改該書的正例段落。

## 子代理分工

- **連貫性檢查**：讀 bible 加前面所有章節，專門挑設定矛盾與斷掉的伏筆。這是機械活，用 Sonnet 5 跑。
- **素讀者**：**不給看 bible 和 outline**，只給成稿，問哪裡看不懂、哪裡無聊、哪個角色動機不成立。讀過設定的代理會自動腦補，看不出資訊缺口——這個限制是刻意的，不要為了「讓它更懂脈絡」而破例。
- 寫稿和結構討論用主模型（Opus 4.8）。
